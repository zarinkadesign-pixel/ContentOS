## П-01: Ежедневная работа директора

```
УТРО (09:00) — 15 минут:
1. Открыть Producer Center → раздел «Входящие»
2. Просмотреть результаты работы агентов за ночь
3. Нажать ✅ Одобрить или ✕ Убрать для каждого результата
4. Проверить CRM — новые лиды из ночного парсинга
5. При необходимости — ответить на входящие сообщения

ВЕЧЕР (21:00) — 10 минут:
1. Проверить статистику публикаций дня
2. Просмотреть аналитику (воскресенье → недельный отчёт)
3. Подтвердить расписание контента на следующий день
```

## П-02: Протокол добавления лида

| Шаг | Действие | Исполнитель |
|----|---------|------------|
| 1 | Лид попадает в CRM (авто из парсинга или вручную) | n8n / Директор |
| 2 | AI генерирует персональное первое сообщение под нишу | Gemini AI |
| 3 | Сообщение отправляется через Telegram/WhatsApp | n8n |
| 4 | Ответил → двигаем: Ответил → Заинтересован | Директор |
| 5 | Заинтересован → предложить созвон (ссылка Calendly) | AI / Директор |
| 6 | После созвона → Контракт или архив | Директор |
| 7 | Контракт подписан → создать клиента в системе | Директор |

**Правило:** Не более 30–50 сообщений в день из Telegram. Интервал между сообщениями — минимум 3 минуты.

## П-03: Протокол онбординга клиента

```
1. Клиент нажимает ссылку на анкету (из SendPulse)
2. Заполняет за 10 минут
3. Данные автоматически → профиль в Producer Center
4. AI Стратег запускается автоматически → первичный анализ за 2 минуты
5. Директор просматривает → одобряет или корректирует
6. AI Распаковщик → brand story, УТП, голос бренда
7. Карта бизнеса активируется → этап 1 «Онбординг» выполнен
```

## П-04: Протокол публикации контента

| Этап | Действие | Инструмент | Срок |
|------|---------|-----------|------|
| Запись | Директор снимает подкаст | Студия / Телефон | 1 день/мес |
| Загрузка | Видео в Google Drive (папка /Подкасты) | Google Drive | Сразу |
| Нарезка | n8n триггер → Vizard API | n8n + Vizard | 10–15 мин |
| Подписи | AI генерирует подписи для каждого клипа | Gemini API | Авто |
| Расписание | Клипы планируются на 3–4 недели | Vizard / Buffer | Авто |
| Публикация | Выходит по расписанию | Buffer / Vizard | Авто |

## П-05: Протокол еженедельной аналитики

```
Каждое воскресенье в 20:00 — n8n запускается автоматически:

1. Собирает метрики: Meta Ads, Instagram API, Producer Center
2. Gemini AI анализирует данные → недельный отчёт
3. Отчёт отправляется директору в Telegram
4. Данные обновляются в профилях клиентов
5. Помечаются слабые результаты:
   • ER < 3% → предупреждение
   • Охват упал > 20% → алерт
   • CTR рекламы < 0.5% → остановить
```

## П-06: Протокол работы с клиентом на продюсировании

```
ЕЖЕМЕСЯЧНЫЙ ЦИКЛ:

Неделя 1:
- AI Копирайтер генерирует тексты для всех постов недели
- AI Планировщик создаёт контент-план
- Директор одобряет → посты выходят автоматически

Неделя 2–3:
- Контент публикуется по расписанию (Buffer / Vizard)
- AI Рекламист оптимизирует рекламные кампании
- AI Аналитик отслеживает метрики еженедельно

Неделя 4 (подготовка к следующему месяцу):
- Созвон с клиентом на запись подкаста
- AI Подкаст-агент готовит 15 вопросов
- Запись → Google Drive → автоматически в Vizard

Конец месяца:
- AI генерирует месячный отчёт для клиента
- n8n отправляет отчёт клиенту
- Директор проводит 30-минутный обзор с клиентом
- AI готовит план на следующий месяц
```

## П-07: Аварийные ситуации

| Ситуация | Признак | Действие |
|---------|--------|---------|
| Telegram заблокирован | Сообщения не доходят | Стоп парсинг, ждать 48ч, снизить лимит до 20/день |
| Vizard ошибка | code ≠ 2000 более 30 мин | Проверить API ключ, повторить запрос вручную |
| Gemini лимит исчерпан | 429 ошибка | Очередь запросов, продолжить на следующий день |
| n8n не запустился | Нет уведомлений | Win+R → cmd → `n8n start`, проверить Task Scheduler |
| Потеря данных платформы | Пустой экран | Настройки → Импорт → выбрать JSON бэкап |
| Buffer не публикует | Посты не выходят | Проверить подключение аккаунтов в Buffer settings |
| Meta реклама остановлена | 0 показов | Проверить баланс, статус объявлений, правила платформы |

## П-08: Протокол завершения договора с клиентом

```
1. Уведомить клиента за 14 дней
2. Подготовить финальный отчёт (AI генерирует)
3. Передать все материалы: тексты, стратегия, пароли
4. Экспортировать данные клиента из платформы (JSON)
5. Перевести профиль клиента в статус «Архив»
6. Данные хранятся 90 дней, затем можно удалить
7. Запросить отзыв и разрешение на кейс
```

## П-09: Протокол еженедельного бэкапа

```
Каждую пятницу в 18:00:
1. Producer Center → Настройки → Экспорт данных (JSON)
2. Сохранить файл в Google Drive (папка /Backups)
3. Сохранить копию локально (папка на ПК)
Итого: 3 копии = платформа + Google Drive + локально
```
