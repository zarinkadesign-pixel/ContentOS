# PRODUCER CENTER — ПРАВИЛА
**AMAImedia.com · Обязательные правила работы v1.0**

---

## PR-01: Правила работы с лидами

1. **НЕ БОЛЕЕ 30–50 сообщений в день** из Telegram. Превышение = блокировка аккаунта на 24–72 часа.
2. **Интервал между сообщениями — минимум 3 минуты.** В n8n обязательно добавить Wait node.
3. **Первое сообщение всегда персонализированное.** AI пишет под нишу конкретного лида. Никаких шаблонов copy-paste.
4. **Никакого generic спама.** Каждое сообщение — как от живого человека, который прочитал профиль.
5. **При отказе — немедленно прекратить контакт** и удалить из активной базы. Не настаивать.
6. **Хранить данные лидов только в Producer Center.** Не в таблицах, не в заметках телефона.
7. **Follow-up только 3 раза:** через 1 день, 3 дня, 7 дней. После третьего — архив без исключений.

## PR-02: Правила работы с клиентами на продюсировании

1. **Данные каждого клиента изолированы.** Агент клиента А не видит данные клиента Б.
2. **Договор подписывается ДО начала работы.** Никаких устных договорённостей.
3. **Ежемесячный отчёт — первое число каждого месяца.** Без исключений. AI генерирует автоматически.
4. **Все изменения стратегии фиксируются** в карте бизнеса клиента в платформе.
5. **Клиент снимает контент сам.** Мы режиссируем, но не снимаем вместо него.
6. **Запись подкаста — раз в месяц, минимум 3 часа.** Это главный источник всего контента.
7. **При завершении договора — данные архивируются, не удаляются** (90 дней после окончания).

## PR-03: Правила создания контента

1. **Все тексты пишутся В ГОЛОСЕ клиента** — тон, стиль, словарь из документа распаковки.
2. **Не публиковать заявленные результаты** без реального кейса с конкретными цифрами.
3. **Рекламные объявления** соответствуют правилам Meta Ads, TikTok, ВКонтакте.
4. **Хэштеги:** максимум 5–7 на пост, только релевантные нише клиента.
5. **CTA в каждом посте — одно чёткое действие.** Не два, не три. Одно.
6. **Кодовое слово для бота — одно.** Не менять без обновления всей воронки.
7. **Визуальный стиль клиента — единый** на всех платформах: цвета, шрифты, фильтры.

## PR-04: Правила работы с API и данными

1. **API ключи хранятся ТОЛЬКО** в настройках Producer Center. Никогда в коде, никогда в переписке.
2. **Регулярный экспорт данных:** каждую пятницу → Настройки → Экспорт JSON → Google Drive.
3. **Три копии данных:** платформа + локальный JSON файл + Google Drive.
4. **При обновлении платформы** — сначала экспорт данных, только потом обновление файла.
5. **Gemini лимит 1,500 запросов/день.** Не запускать более 3 параллельных AI задач.
6. **Vizard:** не более 3 одновременных проектов. Дожидаться завершения перед новым.
7. **n8n при ошибке воркфлоу** — немедленное уведомление в Telegram директору. Не молчать.

## PR-05: Финансовый учёт

1. **Все входящие платежи вносятся в день получения** в раздел Финансы платформы.
2. **30% от дохода клиента** вносится ежемесячно в первых числах месяца.
3. **KPI отслеживается еженедельно**, а не ежемесячно.
4. **Расходы на рекламу фиксируются отдельно** от операционных расходов.
5. **Бонус директора** — 20% от суммы сверх KPI. Выплачивается при достижении $20k/мес.
6. **Инвойсы клиентам** — не позднее 5-го числа каждого месяца.

## PR-06: Лимиты API

| Сервис | Лимит | Действие при превышении |
|--------|-------|------------------------|
| Gemini API | 1,500 запросов/день | Очередь запросов, перенос на следующий день |
| Vizard API | По тарифу | Не > 3 проектов одновременно |
| Telegram (Telethon) | 30–50 сообщений/день | Автостоп в n8n + Telegram alert директору |
| Buffer | 10 постов в очереди (Free) | Публиковать раз в 2 дня, не чаще |
| Phantombuster | 2 часа/месяц (Free) | Только для первичного сбора базы |
| Meta Ads API | По лимитам аккаунта | Проверить Business Manager статус |

## PR-07: Правила работы с Claude Projects

1. **Один проект = один контекст.** Не смешивать бизнес-задачи и личные в одном проекте.
2. **claude.md обновляется** при каждом существенном изменении бизнеса.
3. **Мастер-документация загружается в Files проекта** для глубокого контекста.
4. **Чёткие запросы:** "для клиента Алина, ниша нутрициология, нужен контент-план".
5. **Результаты агентов сохранять** в Заметки или экспортировать сразу.
6. **Директор только одобряет** — не ставит задачи агентам вручную, они инициируют сами.
